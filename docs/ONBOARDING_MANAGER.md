# Онбординг нового менеджера

ADMIN заводит юзера c ролью MANAGER, передаёт ему сгенерированный PIN, менеджер логинится по PIN, привязывает Telegram через deeplink и начинает получать push'и (сводки, inbox-события).

## Предусловия

- ADMIN залогинен в CRM
- У менеджера есть аккаунт в Telegram

## Шаг 1. Создать User в CRM

**Где:** `/settings/users` → кнопка «Добавить пользователя».
**Заголовок модалки:** «Новый пользователь».
**Поля:**
- «Имя» (обязательно)
- «Роль» — выпадайка: «Менеджер» / «Администратор» / «Шеф» / «Курьер»; по умолчанию «Менеджер».

**Кнопка подтверждения:** «Создать».
**PIN автогенерируется** (4 цифры) и **показывается один раз** после создания. Запомните / скопируйте сразу — второй раз он не покажется, можно будет только перегенерировать.
**Что проверить:** новый юзер появился в таблице на `/settings/users`.

## Шаг 2. Передать PIN менеджеру

Канал: лично или защищённый мессенджер (Signal / Telegram-secret-chat).
⚠️ **НЕ в общий чат.** PIN — это пароль; общий чат = leak.

## Шаг 3. Менеджер логинится в CRM

URL: `https://crm-budni.pro` (или `https://budni-crm.vercel.app`).
Ввод PIN на `/login`.
**Что проверить:** после успешного логина попадает на `/dashboard`.

## Шаг 4. Привязка Telegram

Менеджер открывает `/settings/telegram`.
Жмёт **«Сгенерировать ссылку»** — появляется кнопка **«Открыть в Telegram»** (deeplink вида `https://t.me/<TELEGRAM_BOT_USERNAME>?start=<telegramOnboardingToken>`).
По кнопке открывается чат с ботом → нажать `/start` (или предложенную кнопку Start в боте).

Если ссылка устарела или нужна новая — рядом есть кнопка **«Сгенерировать новую»**.

## Шаг 5. Проверка привязки

- **Сам менеджер** на `/settings/telegram` видит «Telegram привязан» и кнопку «Отвязать».
- **ADMIN** на `/settings/users` в колонке «Telegram» у этого менеджера видит зелёный значок и `@username` (если username в Telegram задан) или просто индикатор «привязан». Без привязки в колонке стоит «—».

## Шаг 6. Smoke-тест push'а

Дождаться очередной сводки (см. cron'ы — обычно 14:00 / 15:30 МСК) **или** спровоцировать inbox-событие (например, попросить клиента отправить сообщение MAX-боту → должен прилететь push в Telegram менеджеру).

## Возможные проблемы

- **Deeplink не открывает бота** — проверь `TELEGRAM_BOT_USERNAME` в env (без `@`, только username); если бот недавно поменян — webhook мог не обновиться, см. `npm run telegram:setup-webhook`.
- **Push'и не приходят** — `telegramChatId` у юзера null (привязка не завершилась — попроси повторить `/start`); либо менеджер заблокировал бота в Telegram.
- **Срок действия onboarding-токена истёк** — менеджер на `/settings/telegram` жмёт «Сгенерировать новую».
- **Менеджер забыл PIN** — ADMIN заходит в `/settings/users`, перегенерирует PIN (если такая кнопка есть) или создаёт нового юзера и удаляет старого.
