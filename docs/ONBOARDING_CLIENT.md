# Онбординг нового клиента

Заводим клиента (юрлицо) в CRM, привязываем точку доставки и конфиг питания, подключаем клиента в MAX-боте, чтобы он мог получать вопросы про порции и отвечать боту. Выполняет ADMIN или MANAGER. Занимает 10–15 минут, ещё столько же — на smoke-тест.

## Предусловия

- залогинен как ADMIN или MANAGER
- у клиента есть аккаунт в MAX-мессенджере
- известны реквизиты юрлица, адрес точки, типы питания (завтрак / обед / ужин), кол-во порций, расписание (DAILY / WEEKDAYS / WEEKENDS / CUSTOM_DAYS / INTERVAL / ONE_TIME), цена

## Шаг 1. Создать Client (юрлицо)

**Где:** `/clients/new` (кнопка «Новый клиент» из `/clients`).
**Заголовок страницы:** «Новый клиент».
**Поля:**
- «Название клиента \*» (обязательно)
- «Контактное лицо»
- «Телефон»
- «Мессенджер»
- «Заметки»
- (опционально, в том же шаге) первая точка: «Название точки», «Адрес», «Окно с», «Окно до», «Упаковка»

**Кнопка подтверждения:** «Создать клиента».
**Что проверить:** после успеха клиент виден в списке `/clients`.

## Шаг 2. Создать ClientLocation (точку доставки)

**Где:** `/clients/[id]` → вкладка «Точки · N» → кнопка «Добавить точку».
**Заголовок модалки:** «Новая точка».
**Поля:**
- «Название точки»
- «Адрес»
- «Окно с» (HH:MM)
- «Окно до» (HH:MM)
- «Упаковка» (выбор: «Порционно» / «Коробками»)
- «Пометки» (теги)

**Кнопка подтверждения:** «Создать».

> ⚠️ Если у клиента несколько точек — создайте их все сейчас. После создания ClientMealConfig добавить точку всё ещё можно, но в существующих конфигах `locationId` придётся проставлять вручную (см. «Возможные проблемы»).

## Шаг 3. Создать ClientMealConfig

**Где:** `/clients/[id]` → вкладка «Питание · N» → кнопка «Добавить конфиг».
**Заголовок модалки:** «Новый конфиг питания».
**Поля:**
- «Точка \*» — выпадайка «Выберите локацию…». **Поле обязательное** (с 5.9b `locationId` стал обязательным). Если у клиента одна локация — она preselect'ится, если две и больше — выбрать руками.
- «Типы питания» — чекбоксы «Завтрак» / «Обед» / «Ужин» (можно несколько).
- «Тип заказа» — «Фикс (постоянное число)» или «Динамика (подтверждение)».
- «Цена за порцию, ₽» — отдельно для каждого выбранного типа питания.
- «Количество порций» — при FIXED.
- «Доставка» — «На следующий день» или «В тот же день».
- «График» — «Каждый день» / «Будни (Пн-Пт)» / «Выходные (Сб-Вс)» / «Свои дни недели» / «Разовый заказ» / «С интервалом».
- «Дни недели» — при CUSTOM_DAYS.
- «Интервал в днях» — при INTERVAL.

**Кнопка подтверждения:** «Создать конфиг» (или «Создать N конфига», если выбрано несколько типов питания — будет создано по конфигу на тип).
**Что проверить:** конфиг появился во вкладке «Питание»; `locationId` заполнен (виден как название точки на карточке конфига).

## Шаг 4. Привязать клиента в MAX

**Где:** `/clients/[id]` → секция «MAX-бот».
**Как:**
1. Жмём «Получить ссылку для онбординга» — генерится deeplink вида `https://max.ru/id503018232259_bot?start=<onboardingToken>`.
2. Кнопкой «Копировать» забираем ссылку.
3. Отправляем клиенту любым каналом.
4. Клиент открывает ссылку в MAX → нажимает Start в боте.

**Что проверить:** после прохождения deeplink клиентом в секции «MAX-бот» появляется индикатор «Привязан · chat_id <значение>» (вместо кнопки генерации).

**Запасной вариант:** если deeplink почему-то не отрабатывает, можно ввести `maxChatId` вручную через расширенные настройки секции и нажать «Сохранить».

## Шаг 5. Smoke-тест входящего сообщения

Попроси клиента отправить любое сообщение в бота → проверь, что оно появилось в `/inbox`.

Альтернатива: дождаться cron `daily-questions` (11:00 МСК) — клиент должен получить вопрос про порции (только для DYNAMIC-конфигов; для FIXED вопросы не задаются).

## Шаг 6. Финальная проверка

- [ ] Client создан, виден в `/clients`
- [ ] ClientLocation создана, видна на странице клиента во вкладке «Точки»
- [ ] ClientMealConfig создан с заполненным `locationId` (название точки видно на карточке конфига)
- [ ] Клиент привязан в MAX (индикатор «Привязан · chat_id …» в секции «MAX-бот»)
- [ ] Тестовое сообщение от клиента дошло в `/inbox`, либо cron `daily-questions` сработал и клиент получил вопрос

## Возможные проблемы

- **Клиент не получает сообщения** — проверь `MAX_BOT_TOKEN` в env, что бот запущен, и что клиент реально прошёл deeplink (в `/clients/[id]` должен быть `chat_id`).
- **Заказы не генерируются автоматически** — `Schedule` пустой/некорректный (CUSTOM_DAYS без дней), `mealType` не тот, или cron 06:00 МСК не отработал.
- **Конфиг без `locationId` (legacy, до 5.9b)** — открой конфиг и проставь точку вручную через редактирование, либо пересоздай.
- **Не открывается deeplink** — клиент должен быть авторизован в MAX-мессенджере с того устройства, с которого открывает ссылку; иначе MAX просит логин и теряет токен.
